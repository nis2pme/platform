## What is NIS2PME?

**NIS2PME** is a multi-tenant **GRC (Governance, Risk & Compliance) SaaS platform** that helps Portuguese SMEs self-assess their compliance with the **NIS2 Directive** (transposed into Portuguese law by the *Regime Jurídico da Cibersegurança*, as amended by *Decreto-Lei n.º 125/2025*) and the **QNRCS 2026** (Quadro Nacional de Referência para a Cibersegurança).

It is built on a **Universal Control Framework (UCF)** capable of representing any regulatory framework, instantiated here for the **107 controls** based on the public consultation of QNRCS 2026.

The platform is offered under a **dual model**:

- **Managed service (SaaS)** — hosted and operated for you.
- **On-premises** — the open-source core, distributed as Docker images under the **AGPLv3** license, that you run on your own infrastructure.

This repository contains the **on-premises open-source core**.

---

## Key concepts

| Concept | What it does |
|---------|--------------|
| **Universal Control Framework (UCF)** | A generic model that can represent any normative framework. Loaded here with the 107 QNRCS 2026 controls; other frameworks can be added without code changes. |
| **Gated Maturity Model (GMM)** | Maturity is unlocked level-by-level: the next level only opens once the sub-requirements of the current level are met, scored with a **weighted compliance formula**. |
| **Prioritised action plan** | Automatically generated by crossing the most common SME vulnerabilities with the controls — it tells you **where to start**. |
| **Evidence management** | Upload and attach encrypted evidence files to each control to support your self-assessment. |

## Features

- 📊 **Maturity dashboard** with radar charts and weighted compliance scoring
- 🧭 **Guided setup wizard** that determines whether you are an *important* or *essential* entity
- 🗂️ **Encrypted evidence** storage (Fernet) attached to controls
- 📑 **PDF reports** generated client-side
- 🔐 **Strong security**: Argon2id password hashing, JWT auth, **2FA/TOTP**, full **audit logging**
- 🌐 **Self-contained deployment**: PostgreSQL + FastAPI + nginx, secrets auto-generated on first boot

## Quick Start

> **Requirements:** a Linux server (Ubuntu 20.04+, Debian 11+, RHEL/Rocky 8+, Fedora, etc.) with **Docker Engine 20.10+** and **Docker Compose v2**. Minimum 2 vCPU / 2 GB RAM / 20 GB disk. Port **80/TCP** (and **443/TCP** for HTTPS).

### Option 1 — One-line install (easiest)

For users who just want it running. This downloads the installer, auto-detects your server IP, generates a secure database password, pulls the images from GHCR and starts everything:

```bash
curl -fsSL https://raw.githubusercontent.com/nis2pme/platform/main/start_nis2pme.sh | bash
```

> 🔎 **Security tip:** piping a script straight into `bash` runs remote code on your server. If you prefer to read it first (recommended for any security-sensitive host):
> ```bash
> curl -fsSL https://raw.githubusercontent.com/nis2pme/platform/main/start_nis2pme.sh -o start_nis2pme.sh
> less start_nis2pme.sh        # inspect it
> sh start_nis2pme.sh
> ```

When it finishes, open the URL it prints (e.g. `https://192.168.1.50`) in your browser to run the **setup wizard**.

> ℹ️ **The one-liner runs without prompts.** Piped into `bash` it has no terminal, so it uses defaults: the language menu is skipped (**Portuguese**) and a **temporary self-signed certificate** is generated (your browser warns on the first visit — this is expected). To be prompted for the language and the TLS option (your own certificate / behind-a-proxy / self-signed), download and run it instead: `sh start_nis2pme.sh`. You can always set or replace the certificate later in the setup wizard.

### Option 2 — Docker Compose (a bit more control)

For users who know a little Docker:

```bash
# 1. Get the compose file
curl -fsSL https://raw.githubusercontent.com/nis2pme/platform/main/docker-compose.yml -o docker-compose.yml

# 2. Create a minimal .env
cat > .env <<'EOF'
APP_URL=https://YOUR_SERVER_IP
DB_PASSWORD=change-me-to-a-long-random-string
TLS_MODE=self-signed
EOF

# 3. Start (images are pulled from GHCR — no build needed)
docker compose up -d
```

> `TLS_MODE` can be `self-signed` (default — a temporary certificate is generated), `custom` (your own certificate + key — see [`.env.example`](.env.example)), or `proxy` (TLS terminated upstream by Cloudflare/Traefik/Nginx; in that case use `APP_URL=http://…`). A `./certs` folder is created automatically.

### Option 3 — Manual `docker run` / build from source

For advanced users. The published images are:

```
ghcr.io/nis2pme/backend:latest
ghcr.io/nis2pme/frontend:latest
```

To build the images yourself instead of pulling them, see [CONTRIBUTING.md](CONTRIBUTING.md) and `docker-compose.build.yml`.

## First access — Setup wizard

After the stack boots, open the printed URL. The wizard guides you through 5 steps:

1. **Company details** — name, sector, size (micro/small/medium). The system determines your required compliance level (important vs. essential entity).
2. **Administrator account** — name, email and password of the main admin user.
3. **Email (SMTP)** (optional) — outbound email for password reset; can be configured later.
4. **HTTPS** — review the active TLS certificate and keep it or upload/replace it with your own.
5. **Consents** — Terms & Conditions, Privacy Policy (GDPR), and the optional update check.

Mandatory two-factor authentication (TOTP) is then enrolled before you reach the platform.

You are then redirected to the **maturity dashboard**.

## Configuration

Most settings are auto-generated on first boot and stored in Docker volumes. The values you may set in `.env` are `APP_URL`, `DB_PASSWORD` and `TLS_MODE` (plus the certificate paths when `TLS_MODE=custom`). Optional settings (custom ports, SMTP for password reset, domain/HTTPS) are documented in [`.env.example`](.env.example) and [MANUAL.md](MANUAL.md).

> ⚠️ **Back up the `nis2pme_pgdata` (database), `nis2pme_uploads` (evidence) and `nis2pme_data` (encryption secrets) volumes regularly.** Losing `nis2pme_data` means encrypted data can no longer be decrypted.
